    Dependency Injection

    Welcome to Dependency injection for Orleans. Dependency Injection for Orleans is based off the Microsoft.Dependency.Injection from ASP.NET CORE

    Constructors

    Orleans uses a class as the fundmental block for the grain. It has a constructor, but typically this constructor is empty with no arguments

    Orleans uses the contrcutor to perform constructor injection (see Mark Seeman), where a given dependency is injected into the grain at(really before) activation time.

    So to inject a necessary dedenpency into your grain, give the grain a contructor taking an argument

      public MyGrain(ISuperCoolServiceClient client)

    Where ISuperCoolServiceClient is a interface to the dependency you are injecting. but it could be anything, an int constant, or string with the name of your current pet

    Now if you try running this grain in a silo, your going to get errors e.g. the silo can start that grain. We need a startup class

    Startup Class

    Now that your grain is dependent on something, it's time to fulfill that dependency. A Startup class is the SEAM that performs that action. The Startup class is invoked by the silo when it starts up.

    Lets create a startup class. In this case we are using the Microsoft.Dependency.Injection classes, but you can use whatever DI container you like

      public class Startup
  {
  public Startup(){}
  
  public IServiceProvider ConfigureServices(IServiceCollection services)
  {
            services.AddSingleton<ISuperCoolServiceClient, SuperCoolServiceClient>(f =>
            {
                var configSection = (SuperCoolServiceConfiguration) ConfigurationManager.GetSection(SuperCoolServiceConfiguration.SectionName);
                var baseUrl = new Uri(configSection.BaseUrl);
                //return the concrete implementation
                return new SuperCoolService(baseUrl);
            });
            
           services.AddTransient<MyGrain>(
                sp => new MyGrain(sp.GetRequiredService()));
    return services.BuildServiceProvider(); 
  }

    Now when MyGrain is activated the ISuperCoolServiceClient is injected into the grain

    Using the Startup class

    Production

    In production this is relatvely straight forward. Make the Startup class be used by the Silo startup

    
              siloHost.NodeConfig.StartupTypeName = typeof(Startup).AssemblyQualifiedName;

    Using the TestCluster

    Build a startup class for the class, or fixture or whatever is running your Testcluster

    
    public class TestStartup
    {
        public TestStartup()
        { }

        public IServiceProvider ConfigureServices(IServiceCollection services)
        {
            services.AddTransient<ISuperCoolServiceClient, ISuperCoolServiceClient>(f =>
            {
                var mock = new Mock();
                mock.Setup(m => m.MyMockedFunc(It.IsAny())).ReturnsAsync("My cats name");

                return mock.Object;
            });

            services.AddTransient<MyGrain>(
                sp => new MyGrain(sp.GetRequiredService<ISuperCoolServiceClient>()));

            return services.BuildServiceProvider();

        }
    }
  


    
        public override TestCluster CreateTestCluster()
        {
			var options = new TestClusterOptions();
			options.ClusterConfiguration.AddMemoryStorageProvider("SuperCool");
			options.ClusterConfiguration.AddSimpleMessageStreamProvider("SuperCoolStream");
            //dependency injection
            options.ClusterConfiguration.UseStartupType<TestStartup>();

            return new TestCluster(options);
        }
