    Dependency Injection

    Welcome to Dependency injection for Orleans. Dependency Injection for Orleans is based off the Microsoft.Dependency.Injection from ASP.NET CORE

    Constructors

    Orleans still uses the notion of a class as the fundmental block for the grain. It has a constructor which executed, but normally this constructor needs to be a empty default contructor with no arguments

    Orleans uses the contrcutor to perform constructor injection (see Mark Seeman), where a given dependyc is injected into the grain at(really before) activation time.

    So to injection a necessary dendpency into your grain, give the grain a contrcutor taking an argument

      public MyGrain(ISuperCoolServiceClient client)

    Where 

    ISuperCoolServiceClient
    is a interface to the dependency you are injecting. but it could be anything, a 
    int
    constant, or the name of your current pet

    Startup Class

    Now that your grain is fully dependent on something, its time to fulfill that dependency. A Startup class is the 

    SEAM
    that performs that action. The Startup class is invoked by the silo when it starts up.

    Lets create a startup class. In this case we are using the Microsoft.Dependency.Injection classes, but you can use whatever DI container you like

      public class Startup
  {
  public Startup(){}
  
  public IServiceProvider ConfigureServices(IServiceCollection services)
  {
            services.AddSingleton(f =>
            {
                var configSection = (SuperCoolServiceConfiguration) ConfigurationManager.GetSection(SuperCoolServiceConfiguration.SectionName);
                var baseUrl = new Uri(configSection.BaseUrl);
                //return the concrete implementation
                return new SuperCoolService(baseUrl);
            });
            
           services.AddTransient(
                sp => new MyGrain(sp.GetRequiredService()));
    return services.BuildServiceProvider(); 
  }
